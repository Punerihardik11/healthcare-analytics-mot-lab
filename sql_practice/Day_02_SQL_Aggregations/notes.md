# Day 02 - SQL Aggregations

## Topics Practiced
- GROUP BY
- COUNT()
- SUM()
- AVG()
- HAVING

## Work Done
- Practiced aggregation queries using healthcare claims data
- Calculated average payments by specialty
- Counted claims by provider and specialty
- Calculated total reimbursements using SUM()
- Learned filtering grouped data using HAVING
- Compared WHERE vs HAVING

## Key Learnings
- GROUP BY converts rows into summarized groups
- COUNT() counts rows inside groups
- SUM() calculates total values
- AVG() calculates average values
- HAVING filters grouped/aggregated data
- WHERE filters rows before grouping

## Important Concept
After GROUP BY:
Columns in SELECT must either:
1. be grouped
OR
2. use aggregate functions

## Realizations
- SQL aggregations feel like real business analytics
- Healthcare analytics relies heavily on summarized data
- GROUP BY is one of the most important SQL concepts
